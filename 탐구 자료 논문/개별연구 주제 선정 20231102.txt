주제1. 
현재 구글에서 사용중인 자동입력방지문자 대안 -> reCAPTCHA 2 & 3

버전 2는 구름이나 소화전, 오토바이 등 그림에서부터 사용자에게 특정 영역을 고르게 하고 이로써 실제 사용자인지 봇인지를 판단한다.
시각장애인 같은 경우에는 소리를 듣고 어떤 단어를 말하고 있는지 텍스트 박스 안에 기재하게 한다.
-> 사용자 입장에서 어떤 식으로 인증을 진행해야 할 지 모호하고, 테스트에 적합하지 않다고 결과가 나오고 실제 사용자가 봇으로 판단될 경우에는 특정 사이트를 사용할 때 불이익이 발생할 수 있다. 또한, 봇일 경우에도 이미지 판단 알고리즘을 통해 테스트를 통과할 경우가 존재한다.

-> 이거 실제로 뚫리는지 찾아보기
1. 버전 2 : 실제로 시스템을 이용해서 뚫리는 걸 확인 가능했다. <An Object Detection based Solver for Google's Image reCAPTCHA v2> 참조
2. 버전 3 : 동일하게 시스템을 이용하면 뚫린다. <Hacking Google reCAPTCHA v3 using Reinforcement Learning> 참조

-> 이를 사용자의 개인정보(로그인할 때 사용하는 정보들?)를 활용하여 ZKP 기술을 도합해 사용자는 본인의 개인정보를 알리지 않고 본인이 실제 사용자임을 사이트에 입증할 수 있다. 
-> 즉, 튜링테스트에 ZKP 기술을 도입할 수 있는 방안이 있을까?

버전 3은 사이트를 사용하고 있을 때 마우스의 동작이나 로그인을 할 때의 동작, 이외의 기타 동작들을 실시간으로 판단하여 현재 사이트에 접속중인 것이 실제 사용자(1.0에 가까울 수록 사용자)인지 봇(0.0에 가까울 수록 봇)인지를 판단한다. 
사용중에 봇이라고 판단이 되면 바로 벤이 된다.
-> 시스템이 잘 못 작동해서 실제 사용자가 봇이라고 판단될 경우에는 즉시 사용이 중단된다..? -> 2단계 검증을 통과해야한다.
